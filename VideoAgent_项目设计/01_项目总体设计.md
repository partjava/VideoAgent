# VideoAgent：基于多智能体协作的一键短视频生成系统设计方案

## 1. 项目名称

**VideoAgent：基于多智能体协作的一键短视频生成系统**

也可以作为论文/报告题目：

**基于 DeepSeek 与通义多模态模型的多智能体短视频自动生成系统设计与实现**

---

## 2. 项目定位

本项目面向短视频内容创作场景，设计并实现一个“一句话生成短视频”的多智能体系统。

用户只需要输入一句自然语言需求，例如：

```text
生成一个 60 秒短视频，主题是“大学生为什么容易拖延”，风格轻松幽默，适合抖音。
```

系统会自动完成：

```text
需求解析
脚本生成
分镜设计
图片生成
图生视频
配音生成
字幕生成
背景音乐匹配
自动剪辑合成
视频质检
最终导出
```

最终输出：

```text
final.mp4        最终视频
cover.png        视频封面
script.txt       视频脚本
storyboard.json  分镜文件
subtitle.srt     字幕文件
publish.txt      发布标题、简介和标签
```

---

## 3. 项目核心思想

本项目不是简单调用一个 AI 接口生成内容，而是将短视频制作流程拆分为多个智能体协作完成。

核心流程为（LangGraph StateGraph 编排 + Send 并行）：

```text
第一阶段（串行）：
用户输入一句话
↓
任务规划 Agent → 脚本 Agent → 分镜 Agent → 对白精修 → 提示词 Agent

第二阶段（并行，LangGraph Send）：
                        ┌── 图片 Agent → 视频 Agent ──┐
     提示词 Agent ─Send──┼─────────────────────────────┼──→ 字幕 Agent
                        └── 配音 Agent ────────────────┘

第三阶段（串行 + 条件分支）：
字幕 Agent → 剪辑 Agent → 质检 Agent ──成功──→ 导出 Agent → MP4
                                  └──失败──→ 回剪辑重试
```

系统遵循一个原则：

```text
能用本地库完成的功能，优先使用本地库；
本地效果差或无法完成的生成能力，再调用大模型 API。
```

## 4. 技术路线总览

### 4.1 模型使用原则

| 功能模块    | 使用方案                 | 是否付费 | 说明                  |
| ------- | -------------------- | ---: | ------------------- |
| 脚本生成    | LangChain + DeepSeek |    是 | 负责文本创作              |
| 分镜生成    | LangChain + DeepSeek |    是 | 负责结构化视频设计           |
| 提示词优化   | LangChain + DeepSeek |    是 | 负责生成图片/视频提示词        |
| 图片生成    | Qwen-Image / 通义图像 / **ComfyUI**  | 是/可选  | 生成镜头图片，ComfyUI 可用本地 GPU |
| 图生视频    | 通义万相 / Wan / Vidu / Doubao / **ComfyUI** | 是/可选 | 将图片转为动态视频，ComfyUI 可用本地 GPU |
| 图像/视频检查 | Qwen-VL，可选           | 是，可选 | 检查画面是否符合分镜          |
| 配音      | edge-tts             |    否 | 免费配音，后期可换 CosyVoice |
| 字幕生成    | Python 本地生成 SRT      |    否 | 按分镜时间生成字幕           |
| 视频剪辑    | MoviePy + FFmpeg     |    否 | 本地合成视频              |
| 背景音乐    | 本地免版权音乐库             |    否 | 按风格匹配音乐             |
| 封面生成    | Pillow               |    否 | 本地生成封面标题图           |
| 数据存储    | MongoDB              |    否 | 保存任务、分镜、素材、日志       |
| 异步任务    | Celery + Redis       |    否 | 后台生成视频              |
| 前端页面    | Vue 3 + Element Plus |    否 | 展示生成流程              |
| 后端接口    | FastAPI              |    否 | 提供系统 API            |
| 多智能体编排 | LangGraph            |    否 | StateGraph 调度 13 个 Agent |

## 7. 生成模式设计

为了控制成本和提高系统稳定性，系统设计三种生成模式。

### 7.1 图文视频模式

这是最低成本模式。

流程：

```text
DeepSeek 生成脚本
↓
DeepSeek 生成分镜
↓
Qwen-Image 生成图片
↓
edge-tts 生成配音
↓
Python 生成字幕
↓
MoviePy + FFmpeg 合成视频
```

特点：

```text
成本低
速度快
稳定性高
适合开发测试和普通生成
```

画面表现方式：

```text
图片轻微放大
图片平移
淡入淡出
字幕动画
背景音乐
镜头转场
```

---

### 7.2 关键镜头动态模式

这是推荐的默认模式。

流程：

```text
DeepSeek 生成脚本和分镜
↓
Qwen-Image 生成所有镜头图片
↓
选择 2~4 个关键镜头调用 Wan 图生视频
↓
普通镜头使用本地镜头运动
↓
edge-tts 配音
↓
MoviePy + FFmpeg 合成视频
```

特点：

```text
成本可控
视频观感明显提升
适合项目展示
```

---

### 7.3 全动态视频模式

这是高质量展示模式。

流程：

```text
DeepSeek 生成脚本和分镜
↓
Qwen-Image 生成所有镜头首帧图
↓
Wan 将所有镜头转为动态视频
↓
edge-tts 或 CosyVoice 生成配音
↓
本地合成字幕、配乐和转场
```

特点：

```text
效果最好
成本最高
适合最终演示作品


## 15. 成本控制策略

本项目遵循以下成本控制策略：

```text
1. 前后端、本地存储、字幕、剪辑、配音优先使用免费库
2. 文本生成只调用 DeepSeek，降低文本生成成本
3. 图片生成使用 Qwen-Image，只在需要生成素材时调用
4. 图生视频默认只用于关键镜头
5. 开发调试阶段允许使用默认图片代替 AI 图片
6. 图生视频失败时自动降级为静态图片镜头
7. 不调用云剪辑、云字幕、云配音等非必要付费服务
```

## 18. 项目创新点

本项目的创新点主要体现在：

```text
1. 将短视频制作流程拆解为多个智能体协作完成
2. 支持从一句话自然语言输入到 MP4 视频成片输出
3. 文本生成与多模态生成分离，DeepSeek 负责文本规划，千问/通义系负责图像与视频生成
4. 本地库与云端模型结合，降低系统成本
5. 引入质检智能体，实现生成结果检查与失败重试
6. 支持图文视频、关键镜头动态、全动态三种生成模式
7. 支持分镜级别的重新生成和人工微调
8. 通过 MongoDB 保存完整生成过程，便于追踪每个 Agent 的执行结果
```

## 21. 系统边界

本系统第一版不做以下功能：

```text
1. 不直接发布到抖音、B站、小红书
2. 不做数字人口播
3. 不做复杂剧情连续人物一致性
4. 不保证电影级视频质量
5. 不处理版权不明的外部素材
6. 不自动抓取网络素材
```

项目定位为：

```text
面向短视频创作流程的自动化生成工具，重点解决从主题到视频初稿的快速生成问题。
```

## 22. 最终项目总结

本项目设计并实现了一个基于多智能体协作的一键短视频生成系统。系统以用户的一句话需求作为输入，通过 DeepSeek 完成任务规划、脚本生成、分镜设计和视觉提示词生成，通过 Qwen-Image 和通义万相/Wan 完成图片与视频素材生成，通过 edge-tts、MoviePy、FFmpeg 等本地工具完成配音、字幕、配乐和视频合成，最终输出完整 MP4 视频。

该系统的核心价值在于：

```text
不是简单地让 AI 回答问题，而是让多个智能体协作完成一个完整的短视频生产流程。
```

系统具备以下特点：

```text
输入简单
流程完整
模块清晰
成本可控
可扩展性强
最终结果可展示
```

最终实现目标：

```text
用户输入一句话，系统自动生成一个包含脚本、分镜、画面、配音、字幕、背景音乐和剪辑效果的完整短视频。

## 23. ComfyUI 本地多模态接入

### 23.1 背景

为降低图片生成和图生视频的 API 成本，新增 ComfyUI 本地多模态引擎接入。用户可在自建 ComfyUI 服务器（本机 WSL 或独立 GPU 服务器）上运行图片生成和视频生成，替代付费的 Qwen-Image 和 Wan/Vidu/Doubao API。

### 23.2 架构切换

```text
用户在前端 ComfyUI 配置页设置：
  COMFYUI_BASE_URL=http://服务器IP:8188
  IMAGE_PROVIDER=comfyui  （或 qwen）
  VIDEO_PROVIDER=comfyui  （或 wan/vidu/doubao）

主流程通过 provider_factory 自动选择：
  ImageAgent → get_image_service()
    ├── qwen → QwenImageService
    └── comfyui → ComfyUIImageService

  VideoAgent → get_video_service()
    ├── wan/vidu/doubao → 对应云端 API
    └── comfyui → ComfyUIVideoService
```

### 23.3 功能特性

- 前端独立配置页：ComfyUI 地址、Provider 切换、Workflow JSON 编辑、节点映射配置
- 支持文生图和图生视频 Workflow
- 支持 Wan2.2 Text-to-Video 等社区模型
- 单独测试生成，不进入主流程
- 6 个预置故事模板 + DeepSeek 辅助生成详细视频提示词
- WebSocket 实时进度追踪
- 测试结果保存并显示在历史任务页面

### 23.4 后端模块

```text
backend/services/comfyui/
├── client.py              # ComfyUI API 客户端（HTTP + WebSocket）
├── config_store.py        # 配置读写（.env + 运行时）
├── workflow_store.py      # Workflow JSON 持久化
├── workflow_mapper.py     # 节点映射注入 + 输出文件提取
├── test_history.py        # 测试结果存储

backend/services/image/comfyui_image_service.py
backend/services/video/comfyui_video_service.py
backend/routes/comfyui_routes.py
```
