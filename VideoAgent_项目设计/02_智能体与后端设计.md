# 8.1 TaskPlannerAgent：任务规划智能体

## 功能定位

TaskPlannerAgent 负责理解用户输入，将一句自然语言需求转化为结构化视频任务配置。

## 输入

```json
{
  "user_input": "生成一个60秒短视频，主题是大学生为什么容易拖延，风格轻松幽默，适合抖音。"
}
```

## 处理内容

TaskPlannerAgent 需要识别：

```text
视频主题
目标时长
视频风格
目标平台
目标受众
画面比例
生成模式
是否需要配音
是否需要字幕
是否需要图生视频
```

## 输出

```json
{
  "topic": "大学生为什么容易拖延",
  "duration": 60,
  "style": "轻松幽默",
  "platform": "抖音",
  "target_audience": "大学生",
  "ratio": "9:16",
  "video_type": "知识科普",
  "voice_enabled": true,
  "subtitle_enabled": true,
  "generation_mode": "key_dynamic"
}
```

## 使用模型

```text
DeepSeek
```

## 设计说明

该智能体是整个系统的入口。
它的作用不是生成最终内容，而是把模糊需求转成后续智能体可以执行的标准任务。

# 8.2 ScriptAgent：脚本生成智能体

## 功能定位

ScriptAgent 负责根据任务配置生成短视频口播脚本。

## 输入

```json
{
  "topic": "大学生为什么容易拖延",
  "duration": 60,
  "style": "轻松幽默",
  "target_audience": "大学生",
  "platform": "抖音"
}
```

## 处理内容

ScriptAgent 需要生成：

```text
视频标题
开头钩子
主体内容
结尾总结
口播脚本
发布文案草稿
```

脚本结构建议：

```text
开头 3~5 秒：吸引注意
中间 40~50 秒：讲清楚核心内容
结尾 5~10 秒：总结或引导
```

## 输出

```json
{
  "title": "大学生拖延，不一定是懒",
  "hook": "你是不是每次准备学习，都会先刷五分钟手机？",
  "script": "你以为拖延是因为懒，其实很多时候是因为任务太大，大脑自动开始逃避。当任务看起来很难时，大脑会倾向于选择更轻松的事情，比如刷手机、整理桌面、喝水。解决拖延的关键不是逼自己马上完成全部，而是把任务拆成一个很小的开始。比如先学五分钟，先写一行字，先打开资料。只要开始了，大脑的压力就会下降，继续做下去就容易多了。",
  "ending": "所以，打败拖延最好的办法不是等状态，而是先开始五分钟。"
}
```

## 使用模型

```text
DeepSeek
```

## 设计说明

该智能体决定视频内容质量。
要求输出内容适合配音，句子不能太长，表达要口语化。

# 8.3 StoryboardAgent：分镜设计智能体

## 功能定位

StoryboardAgent 负责把完整脚本拆分为多个镜头，并为每个镜头生成时间、口播、字幕和画面描述。

## 输入

```json
{
  "script": "你是不是每次准备学习，都会先刷五分钟手机……",
  "duration": 60,
  "style": "轻松幽默",
  "ratio": "9:16"
}
```

## 处理内容

StoryboardAgent 需要完成：

```text
将脚本拆成 6~12 个镜头
确定每个镜头时长
生成每个镜头的口播内容
生成每个镜头的字幕
生成每个镜头的画面描述
判断哪些镜头适合图生视频
```

## 输出

```json
[
  {
    "scene_id": "scene_001",
    "scene_index": 1,
    "duration": 5,
    "voiceover": "你是不是每次准备学习，都会先刷五分钟手机？",
    "subtitle": "准备学习，却先刷手机？",
    "visual_description": "一名大学生坐在书桌前准备学习，旁边手机不断弹出消息通知",
    "camera_motion": "slow_zoom_in",
    "need_dynamic_video": true
  },
  {
    "scene_id": "scene_002",
    "scene_index": 2,
    "duration": 6,
    "voiceover": "你以为拖延是因为懒，其实很多时候是因为任务太大。",
    "subtitle": "拖延不一定是懒",
    "visual_description": "学生看着满屏任务清单，表情崩溃，画面轻松幽默",
    "camera_motion": "pan_left",
    "need_dynamic_video": false
  }
]
```

## 使用模型

```text
DeepSeek
```

## 设计说明

该智能体是连接文本和视频的关键。
后续的图片生成、图生视频、字幕、剪辑都依赖分镜数据。

# 8.4 PromptAgent：视觉提示词智能体

## 功能定位

PromptAgent 负责将分镜中的中文画面描述转化为适合图片生成和视频生成模型使用的提示词。

## 输入

```json
{
  "scene_id": "scene_001",
  "visual_description": "一名大学生坐在书桌前准备学习，旁边手机不断弹出消息通知",
  "style": "轻松幽默",
  "ratio": "9:16"
}
```

## 处理内容

PromptAgent 需要生成：

```text
图片生成提示词
图生视频提示词
负面提示词
统一风格描述
镜头运动描述
```

## 输出

```json
{
  "scene_id": "scene_001",
  "image_prompt": "竖屏9:16，现代卡通插画风格，一名大学生坐在书桌前准备学习，桌面有书本和电脑，旁边手机屏幕不断弹出消息通知，画面轻松幽默，色彩明亮，干净背景，高质量插画",
  "video_prompt": "镜头缓慢推进，手机通知轻微闪烁，学生表情从认真变成分心，整体节奏轻松自然",
  "negative_prompt": "模糊，低清晰度，畸形手指，文字乱码，多余肢体，画面拥挤，比例错误"
}
```

## 使用模型

```text
DeepSeek
```

## 设计说明

图片和视频生成效果很大程度取决于提示词质量。
PromptAgent 的作用是保证不同镜头风格统一，避免每个镜头画风乱跳。

# 8.5 ImageAgent：图片生成智能体

## 功能定位

ImageAgent 负责调用千问/通义图像模型，为每个分镜生成图片素材。

## 输入

```json
{
  "scene_id": "scene_001",
  "image_prompt": "竖屏9:16，现代卡通插画风格，一名大学生坐在书桌前准备学习……",
  "negative_prompt": "模糊，低清晰度，畸形手指……",
  "ratio": "9:16"
}
```

## 处理内容

ImageAgent 需要完成：

```text
调用 Qwen-Image / 通义图像接口
生成镜头图片
保存图片到本地目录
记录图片路径
失败时自动重试
```

## 输出

```json
{
  "scene_id": "scene_001",
  "asset_type": "image",
  "asset_path": "/assets/task_001/images/scene_001.png",
  "status": "success"
}
```

## 使用模型

```text
Qwen-Image / 通义图像模型
```

## 设计说明

该智能体属于多模态生成模块。
如果图片生成失败，需要自动重试，或者使用备用提示词重新生成。

# 8.6 VideoAgent：图生视频智能体

## 功能定位

VideoAgent 负责将部分或全部镜头图片转化为动态视频片段。

## 输入

```json
{
  "scene_id": "scene_001",
  "image_path": "/assets/task_001/images/scene_001.png",
  "video_prompt": "镜头缓慢推进，手机通知轻微闪烁，学生表情从认真变成分心",
  "duration": 5,
  "resolution": "720P"
}
```

## 处理内容

VideoAgent 需要完成：

```text
判断当前镜头是否需要图生视频
调用通义万相 / Wan 图生视频接口
生成 3~5 秒视频片段
保存视频片段
记录视频路径
失败时降级为静态图片镜头
```

## 输出

```json
{
  "scene_id": "scene_001",
  "asset_type": "video",
  "asset_path": "/assets/task_001/videos/scene_001.mp4",
  "duration": 5,
  "status": "success"
}
```

## 使用模型

```text
通义万相 / Wan 图生视频模型
```

## 设计说明

VideoAgent 是增强视频观感的核心模块。
为了控制成本，系统可以只对关键镜头调用该智能体，其余镜头由本地剪辑 Agent 使用图片运动效果实现。

# 8.7 VoiceAgent：配音智能体

## 功能定位

VoiceAgent 负责将每个分镜的配音文本转化为独立的配音音频文件。

## 输入

```json
{
  "scenes": [
    {
      "scene_id": "scene_001",
      "voiceover": "你是不是每次准备学习，都会先刷五分钟手机？",
      "duration": 5
    }
  ]
}
```

## 处理内容

VoiceAgent 需要完成：

```text
清洗每个分镜的配音文本（去说话人标签）
为每个分镜单独调用 TTS 生成 voice_{scene_id}.mp3
保存每个分镜配音到 assets 集合（带 scene_id 标记）
更新 scenes 集合中每个分镜的 audio_path 和 audio_duration
同时生成完整的 voice.mp3（用于前端预览）
不再覆盖分镜时长，视频总时长以分镜规划的原始时长为准
```

## 输出

```json
{
  "voice_path": "/assets/task_001/audio/voice.mp3",
  "per_scene": [
    {"scene_id": "scene_001", "path": "/assets/task_001/audio/voice_scene_001.mp3"},
    {"scene_id": "scene_002", "path": "/assets/task_001/audio/voice_scene_002.mp3"}
  ],
  "duration": 58.6,
  "status": "success"
}
```

## 默认方案

```text
edge-tts
```

## 可选升级方案

```text
CosyVoice
```

## 设计说明

第一版建议使用 edge-tts，因为它不花钱，工程实现简单。
VoiceAgent 不再根据配音时长覆盖分镜时长，保证视频总时长以分镜规划为准。
每个分镜生成独立配音文件，供 EditorAgent 按分镜蒙太奇合成。
如果后续对配音效果要求更高，可以切换为阿里 CosyVoice。

# 8.8 SubtitleAgent：字幕智能体

## 功能定位

SubtitleAgent 负责根据分镜和配音内容生成字幕文件。

## 输入

```json
{
  "scenes": [
    {
      "scene_index": 1,
      "duration": 5,
      "subtitle": "准备学习，却先刷手机？"
    }
  ]
}
```

## 处理内容

SubtitleAgent 需要完成：

```text
根据每个镜头时长计算字幕开始时间和结束时间
生成 SRT 字幕文件
保存 subtitle.srt
检查字幕是否为空
检查字幕长度是否过长
```

## 输出

```srt
1
00:00:00,000 --> 00:00:05,000
准备学习，却先刷手机？

2
00:00:05,000 --> 00:00:11,000
拖延不一定是懒
```

## 使用技术

```text
Python 本地生成
```

## 设计说明

第一版不需要语音识别对齐，直接根据分镜时间生成字幕即可。
后期可以接入 Whisper 或其他语音识别工具，实现更精准的字幕时间轴。

# 8.9 MusicAgent：背景音乐智能体

## 功能定位

MusicAgent 负责根据视频风格选择合适的背景音乐，并调整音量和时长。

## 输入

```json
{
  "style": "轻松幽默",
  "duration": 60
}
```

## 处理内容

MusicAgent 需要完成：

```text
从本地音乐库选择匹配音乐
裁剪音乐长度
调整音乐音量
避免盖过配音
输出 bgm.mp3
```

## 输出

```json
{
  "music_path": "/assets/music/light.mp3",
  "volume": 0.18,
  "status": "success"
}
```

## 使用技术

```text
本地免版权音乐库
MoviePy / FFmpeg
```

## 设计说明

背景音乐不建议调用付费模型生成。
本地准备几首不同风格的免版权音乐即可。

# 8.10 EditorAgent：自动剪辑智能体

## 功能定位

EditorAgent 负责将图片、视频片段、配音、字幕和背景音乐合成为最终短视频。

## 输入

```json
{
  "scenes": [
    {
      "scene_id": "scene_001",
      "asset_path": "/assets/task_001/videos/scene_001.mp4",
      "duration": 5,
      "subtitle": "准备学习，却先刷手机？",
      "audio_path": "/assets/task_001/audio/voice_scene_001.mp3",  // 分镜独立配音
      "camera_motion": "slow_zoom_in"
    }
  ],
  "subtitle_path": "/assets/task_001/subtitles/subtitle.srt",
  "music_path": "/assets/music/light.mp3",
  "ratio": "9:16"
}
```

## 处理内容

EditorAgent 需要完成：

```text
统一所有素材尺寸
处理竖屏比例 9:16
将图片转为视频片段
对静态图片添加镜头运动效果
遍历分镜，为每个分镜加载独立配音（scene_audio_map）
将分镜配音挂载到对应 clip 上
拼接所有镜头
添加转场
烧录字幕或外挂字幕
导出 final.mp4
```

## 输出

```json
{
  "video_path": "/outputs/task_001/final.mp4",
  "duration": 59.2,
  "status": "success"
}
```

## 使用技术

```text
MoviePy
FFmpeg
```

## 设计说明

EditorAgent 是系统工程实现的核心。
它不需要调用付费 API，全部使用本地库完成。
EditorAgent 不再使用单一的 voice.mp3 作为整段视频配乐，
而是从 assets 集合中读取每个分镜的独立配音（scene_id 标记），
把配音挂载到对应分镜的 clip 上，避免因配音长度裁剪视频总时长。

# 8.11 QualityAgent：质检智能体

## 功能定位

QualityAgent 负责检查生成结果是否合格，并判断是否需要重新生成部分内容。

## 输入

```json
{
  "task_id": "task_001",
  "video_path": "/outputs/task_001/final.mp4",
  "scenes": [],
  "assets": [],
  "target_duration": 60
}
```

## 处理内容

QualityAgent 需要检查：

```text
视频文件是否存在
视频是否可以正常播放
总时长是否接近目标时长
所有镜头素材是否完整
配音是否存在
字幕是否存在
字幕是否为空
视频比例是否正确
是否有生成失败的镜头
是否需要重新生成某些镜头
```

## 规则质检输出

```json
{
  "status": "passed",
  "duration": 59.2,
  "ratio": "9:16",
  "missing_assets": [],
  "problems": []
}
```

## 失败输出

```json
{
  "status": "failed",
  "problems": [
    "scene_003 图片生成失败",
    "字幕文件为空",
    "视频总时长只有 42 秒，低于目标时长"
  ],
  "next_actions": [
    "重新生成 scene_003 图片",
    "重新生成字幕文件",
    "重新执行视频合成"
  ]
}
```

## 使用技术

```text
Python 规则检查
FFmpeg 获取视频信息
DeepSeek 生成修改建议
Qwen-VL 可选用于画面一致性检查
```

## 设计说明

该智能体体现系统的闭环能力。
它不是简单生成视频，而是会检查生成结果，并在失败时返回对应智能体重新处理。

## 9. Agent 协作流程

### 9.1 标准生成流程（LangGraph StateGraph 编排）

```text
第一阶段（串行）：
1. 用户提交一句话需求
2. TaskPlannerAgent 解析任务
3. ScriptAgent 生成脚本
4. StoryboardAgent 生成分镜
5. DialoguePolishAgent 精修对白
6. PromptAgent 生成图片和视频提示词

第二阶段（并行，LangGraph Send 分叉 + 屏障汇聚）：
7. ImageAgent 生成图片 ──→ 8. VideoAgent 生成视频 ──┐
                                                        ├──→ 10. SubtitleAgent
9. VoiceAgent 配音（无依赖）──────────────────────┘

第三阶段（串行 + 条件分支）：
11. EditorAgent 合成视频
12. QualityAgent 检查
    ├── 通过 → 14. ExportAgent → 完成
    └── 失败 → 回 11. EditorAgent 重试
```

---

### 9.2 失败重试流程

```text
图片生成失败
→ ImageAgent 重新生成
→ 若再次失败，使用默认占位图

图生视频失败
→ VideoAgent 重试
→ 若失败，降级为静态图片镜头

配音失败
→ VoiceAgent 重试
→ 若失败，提示用户重新生成

视频合成失败
→ EditorAgent 检查素材
→ 修复素材路径或格式后重新合成

质检失败
→ QualityAgent 给出问题
→ 返回对应 Agent 重新执行
```

## 11. 后端接口设计

### 11.1 创建任务

```http
POST /api/video/create
```

请求参数：

```json
{
  "user_input": "生成一个60秒短视频，主题是大学生为什么容易拖延，风格轻松幽默",
  "duration": 60,
  "style": "轻松幽默",
  "ratio": "9:16",
  "generation_mode": "key_dynamic"
}
```

返回：

```json
{
  "task_id": "task_001",
  "status": "created"
}
```

---

### 11.2 查询任务进度

```http
GET /api/video/{task_id}/progress
```

返回：

```json
{
  "task_id": "task_001",
  "status": "rendering",
  "progress": 70,
  "current_agent": "EditorAgent",
  "message": "正在合成最终视频"
}
```

```http
GET /api/video/{task_id}
```

```json
{
  "task_id": "task_001",

  "status": "success",
  "scenes": [],

```

## 14. 核心工作流状态设计

任务状态可以设计为：

```text
created          已创建
planning         正在解析任务
script_done      脚本生成完成
storyboard_done  分镜生成完成
image_generating 正在生成图片
video_generating 正在生成视频片段
voice_generating 正在生成配音
editing          正在剪辑合成
checking         正在质检
success          生成成功
failed           生成失败
```

每个 Agent 执行后更新任务状态和进度。
