## 23. 前端展示效果设计

本系统前端主要用于展示“一句话生成短视频”的完整流程。前端不只是一个简单输入框，而是需要清晰展示每个 Agent 的执行状态、生成内容、分镜结果、素材预览和最终视频效果。

前端页面整体风格建议采用简洁科技风，突出“AI 自动生成”“多智能体协作”“视频生成进度”这几个特点。

---

# 23.1 前端整体布局

系统前端采用左侧导航栏 + 右侧内容区的布局。

```text
┌──────────────────────────────────────────────┐
│ 顶部栏：VideoAgent 一键短视频生成系统          │
├──────────────┬───────────────────────────────┤
│ 左侧菜单      │ 右侧主要内容区域                │
│              │                               │
│ 创建视频      │ 根据当前页面展示不同内容          │
│ 任务进度      │                               │
│ 分镜编辑      │                               │
│ 视频预览      │                               │
│ 历史任务      │                               │
│ 系统设置      │                               │
└──────────────┴───────────────────────────────┘
```

左侧菜单包含：

```text
1. 创建视频
2. 任务进度
3. 分镜编辑
4. 视频预览
5. 历史任务
6. 系统设置
```

这也是本项目区别于普通视频生成工具的重要设计。

# 24.17 UI 交互状态

所有生成任务都需要有明确交互状态。

## 按钮状态

```text
默认状态
悬浮状态
点击状态
加载状态
禁用状态
```

例如“开始生成视频”按钮：

```text
默认：开始生成视频
加载：正在创建任务...
禁用：生成中不可重复点击
完成：任务已创建
```

## 生成状态

```text
等待中
生成中
生成成功
生成失败
重试中
已跳过
```

每种状态都要有颜色区分。

# 24.18 空状态设计

如果用户还没有生成任务，历史任务页显示：

```text
暂无视频任务
点击“创建视频”开始生成你的第一个短视频
```

素材为空时显示：

```text
素材尚未生成
请等待 ImageAgent 执行完成
```

视频为空时显示：

```text
视频尚未合成
请等待 EditorAgent 执行完成
```

# 24.19 错误提示设计

当接口调用失败时，前端需要显示明确错误。

示例：

```text
图片生成失败，请检查通义 API Key 或稍后重试。
```

```text
视频合成失败，可能是素材缺失，请前往质检页面查看详情。
```

错误提示不要只显示：

```text
Error
```

而要显示用户能理解的原因和下一步操作。

# 24.20 前端 UI 总结

本系统前端 UI 设计的核心不是复杂动画，而是清晰展示：

```text
用户输入了什么
系统生成到了哪一步
每个 Agent 正在做什么
每一步生成了什么结果
最终视频是否合格
用户可以如何修改和重新生成
```

因此，前端 UI 应围绕以下五个重点设计：

```text
1. 一句话创建任务
2. 多 Agent 流程可视化
3. 分镜卡片化编辑
4. 素材和视频预览
5. 质检结果和日志可追踪
```

这样可以让用户直观看到：

```text
一句话如何一步一步变成完整短视频。
```
