# VideoAgent：基于多智能体协作的一键短视频生成系统设计方案

## 1. 项目名称

**VideoAgent：基于多智能体协作的一键短视频生成系统**

## 2. 项目定位

本项目面向短视频内容创作场景，实现"一句话生成短视频"的多智能体系统。

用户只需输入一句自然语言需求，系统自动完成需求解析、脚本生成、分镜设计、图片生成、图生视频、配音生成、字幕生成、自动剪辑合成、视频质检、最终导出，最终输出 final.mp4、分镜文件、字幕文件和发布文案。

---

## 3. 项目核心思想

本项目不是简单调用一个 AI 接口，而是将短视频制作流程拆分为多个智能体，使用 **LangGraph StateGraph** 编排协作完成。

核心流程：

```
第一阶段（串行）：
用户输入一句话
↓
任务规划 Agent → 脚本 Agent → 分镜 Agent → 对白精修 → 提示词 Agent

第二阶段（并行，LangGraph Send）：
                        ┌── 图片 Agent → 视频 Agent ──┐
     提示词 Agent ─Send──┼─────────────────────────────┼──→ 字幕 Agent
                        └── 配音 Agent ────────────────┘

第三阶段（串行 + 条件分支）：
字幕 Agent → 剪辑 Agent → 质检 Agent ──成功──→ 导出 Agent → MP4
                                  └──失败──→ 回剪辑重试
```

## 4. 技术路线总览

### 4.1 模型使用原则

| 功能模块 | 使用方案 | 是否付费 | 说明 |
| ------- | ------- | ------- | ---- |
| 脚本生成 | DeepSeek | 是 | 负责文本创作 |
| 分镜生成 | DeepSeek | 是 | 负责结构化视频设计 |
| 提示词优化 | DeepSeek | 是 | 负责生成图片/视频提示词 |
| 图片生成 | Doubao Seedream 5.0 Lite | 是 | 火山引擎方舟平台，替代旧 Qwen-Image |
| 图生视频 | Doubao Seedance 1.5 Pro | 是 | 火山引擎方舟平台，替代旧 Wan |
| 配音 | edge-tts | 否 | 免费配音，每个分镜独立配音文件 |
| 字幕生成 | Python 本地生成 SRT | 否 | 按分镜时间生成字幕 |
| 视频剪辑 | MoviePy + FFmpeg | 否 | 本地合成视频，音画自动对齐 |
| 数据存储 | MongoDB | 否 | 保存任务、分镜、素材、日志 |
| 前端页面 | Vue 3 + Element Plus | 否 | 展示生成流程 |
| 后端接口 | FastAPI + LangGraph | 否 | 提供系统 API 和 Agent 编排 |

## 6. Agent 协作流程

### 6.1 标准生成流程

```
1. 用户提交一句话需求
2. TaskPlannerAgent 解析任务（调 DeepSeek）
3. ScriptAgent 生成脚本（调 DeepSeek）
4. StoryboardAgent 生成分镜（调 DeepSeek，每个分镜至少 4 秒）
5. DialoguePolishAgent 精修对白（调 DeepSeek）
6. PromptAgent 生成图片和视频提示词（调 DeepSeek）
7. ImageAgent 生成所有镜头图片（调 Doubao Seedream，4 个并发）
8. VoiceAgent 生成每个分镜独立配音（edge-tts，与图片并行）
9. VideoAgent 对动态镜头生成视频（调 Doubao Seedance，3 个并发）
   - 被内容审核拦截时 → 调 DeepSeek 重写提示词 → 重试（最多 2 次）
   - 失败降级为静态图片
10. SubtitleAgent 生成字幕
11. EditorAgent 合成视频（每个分镜挂载独立配音，音频超视频时长自动裁剪）
12. QualityAgent 检查视频结果
13. ExportAgent 导出最终结果
```

### 6.2 LangGraph 并行编排

使用 LangGraph 的 `Send()` 实现分支并行：

```
prompts
  ├─ images → video  (分支 A，顺序执行)
  └─ voice           (分支 B，与 A 同时跑)
       ↓         ↓
      subtitle (屏障汇聚，等待 A+B 都完成)
```

## 7. Agent 详细设计

### 7.1 同步调用处理

所有调用 DeepSeek 等外部 API 的操作均使用 `anyio.to_thread.run_sync()` 封装，避免阻塞 asyncio 事件循环。Pipeline 运行时其他 API 请求（如进度查询、卡片数据加载）可正常响应。

### 7.2 图片生成（ImageAgent）

- 最多 4 张图片并发生成
- 支持横屏/竖屏比例（`2848x1600` 或 `1600x2848`）
- Seedream 5.0 Lite 最低像素要求 3,686,400

### 7.3 视频生成（VideoAgent）

- 最多 3 个视频并发生成
- Seedance 1.5 Pro 支持 4~12 秒，不足自动补到 4 秒
- 内容审核拦截时自动检测 `sensitive information` 错误
- 调 DeepSeek 重写 motion_prompt（暴力词替换为中性动作描述）
- 最多重写 2 次，失败才降级静态图

### 7.4 视频合成（EditorAgent）

- 每个分镜从 assets 集合读取独立配音文件
- 配音时长超出视频时长时 **加速语音播放**（`vfx.speedx`），保持视频画面原速，不裁剪音频
- 烧录 SRT 字幕
- 去掉 AI 视频原声，使用独立配音音轨

## 11. 失败重试策略

| 失败类型 | 处理方式 |
|----------|----------|
| 图片生成失败 | ImageAgent 重试 |
| 视频生成-内容审核拦截 | DeepSeek 重写提示词 → 重试（最多 2 次） |
| 视频生成-其他错误 | 降级为静态图片 |
| 配音失败 | VoiceAgent 重试 |
| 质检失败 | 回 EditorAgent 重新合成 |
| Pipeline 异常 | 标记任务失败，记录错误日志 |
