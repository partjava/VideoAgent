# VideoAgent：基于多智能体协作的一键短视频生成系统设计方案

## 1. 项目名称

**VideoAgent：基于多智能体协作的一键短视频生成系统**

也可以作为论文/报告题目：

**基于 DeepSeek 与通义多模态模型的多智能体短视频自动生成系统设计与实现**

---

## 2. 项目定位

本项目面向短视频内容创作场景，设计并实现一个“一句话生成短视频”的多智能体系统。

用户只需要输入一句自然语言需求，例如：

```text
生成一个 60 秒短视频，主题是“大学生为什么容易拖延”，风格轻松幽默，适合抖音。
```

系统会自动完成：

```text
需求解析
脚本生成
分镜设计
图片生成
图生视频
配音生成
字幕生成
背景音乐匹配
自动剪辑合成
视频质检
最终导出
```

最终输出：

```text
final.mp4        最终视频
cover.png        视频封面
script.txt       视频脚本
storyboard.json  分镜文件
subtitle.srt     字幕文件
publish.txt      发布标题、简介和标签
```

---

## 3. 项目核心思想

本项目不是简单调用一个 AI 接口生成内容，而是将短视频制作流程拆分为多个智能体协作完成。

核心流程为：

```text
第一阶段（串行）：
用户输入一句话
↓
任务规划 Agent → 脚本 Agent → 分镜 Agent → 对白精修 → 提示词 Agent

第二阶段（并行，LangGraph Send）：
                        ┌── 图片 Agent → 视频 Agent ──┐
     提示词 Agent ─Send──┼─────────────────────────────┼──→ 字幕 Agent
                        └── 配音 Agent ────────────────┘

第三阶段（串行 + 条件分支）：
字幕 Agent → 剪辑 Agent → 质检 Agent ──成功──→ 导出 Agent → MP4
                                  └──失败──→ 回剪辑重试
```

系统遵循一个原则：

```text
能用本地库完成的功能，优先使用本地库；
本地效果差或无法完成的生成能力，再调用大模型 API。
```

## 7. 生成模式设计

为了控制成本和提高系统稳定性，系统设计三种生成模式。

### 7.1 图文视频模式

这是最低成本模式。

流程：

```text
DeepSeek 生成脚本
↓
DeepSeek 生成分镜
↓
Qwen-Image 生成图片
↓
edge-tts 生成配音
↓
Python 生成字幕
↓
MoviePy + FFmpeg 合成视频
```

特点：

```text
成本低
速度快
稳定性高
适合开发测试和普通生成
```

画面表现方式：

```text
图片轻微放大
图片平移
淡入淡出
字幕动画
背景音乐
镜头转场
```

---

### 7.2 关键镜头动态模式

这是推荐的默认模式。

流程：

```text
DeepSeek 生成脚本和分镜
↓
Qwen-Image 生成所有镜头图片
↓
选择 2~4 个关键镜头调用 Wan 图生视频
↓
普通镜头使用本地镜头运动
↓
edge-tts 配音
↓
MoviePy + FFmpeg 合成视频
```

特点：

```text
成本可控
视频观感明显提升
适合项目展示
```

---

### 7.3 全动态视频模式

这是高质量展示模式。

流程：

```text
DeepSeek 生成脚本和分镜
↓
Qwen-Image 生成所有镜头首帧图
↓
Wan 将所有镜头转为动态视频
↓
edge-tts 或 CosyVoice 生成配音
↓
本地合成字幕、配乐和转场
```

特点：

```text
效果最好
成本最高
适合最终演示作品


## 15. 成本控制策略

本项目遵循以下成本控制策略：

```text
1. 前后端、本地存储、字幕、剪辑、配音优先使用免费库
2. 文本生成只调用 DeepSeek，降低文本生成成本
3. 图片生成使用 Qwen-Image，只在需要生成素材时调用
4. 图生视频默认只用于关键镜头
5. 开发调试阶段允许使用默认图片代替 AI 图片
6. 图生视频失败时自动降级为静态图片镜头
7. 不调用云剪辑、云字幕、云配音等非必要付费服务
```

本项目设计并实现了一个基于多智能体协作的一键短视频生成系统。系统以用户的一句话需求作为输入，通过 DeepSeek 完成任务规划、脚本生成、分镜设计和视觉提示词生成，通过 Qwen-Image 和通义万相/Wan 完成图片与视频素材生成，通过 edge-tts、MoviePy、FFmpeg 等本地工具完成配音、字幕、配乐和视频合成，最终输出完整 MP4 视频。

# 8.1 TaskPlannerAgent：任务规划智能体

## 功能定位

TaskPlannerAgent 负责理解用户输入，将一句自然语言需求转化为结构化视频任务配置。

## 输入

```json
{
  "user_input": "生成一个60秒短视频，主题是大学生为什么容易拖延，风格轻松幽默，适合抖音。"
}
```

## 处理内容

TaskPlannerAgent 需要识别：

```text
视频主题
目标时长
视频风格
目标平台
目标受众
画面比例
生成模式
是否需要配音
是否需要字幕
是否需要图生视频
```

## 输出

```json
{
  "topic": "大学生为什么容易拖延",
  "duration": 60,
  "style": "轻松幽默",
  "platform": "抖音",
  "target_audience": "大学生",
  "ratio": "9:16",
  "video_type": "知识科普",
  "voice_enabled": true,
  "subtitle_enabled": true,
  "generation_mode": "key_dynamic"
}
```

## 使用模型

```text
DeepSeek
```

## 设计说明

该智能体是整个系统的入口。
它的作用不是生成最终内容，而是把模糊需求转成后续智能体可以执行的标准任务。

---

# 8.2 ScriptAgent：脚本生成智能体

## 功能定位

ScriptAgent 负责根据任务配置生成短视频口播脚本。

## 输入

```json
{
  "topic": "大学生为什么容易拖延",
  "duration": 60,
  "style": "轻松幽默",
  "target_audience": "大学生",
  "platform": "抖音"
}
```

## 处理内容

ScriptAgent 需要生成：

```text
视频标题
开头钩子
主体内容
结尾总结
口播脚本
发布文案草稿
```

脚本结构建议：

```text
开头 3~5 秒：吸引注意
中间 40~50 秒：讲清楚核心内容
结尾 5~10 秒：总结或引导
```

## 输出

```json
{
  "title": "大学生拖延，不一定是懒",
  "hook": "你是不是每次准备学习，都会先刷五分钟手机？",
  "script": "你以为拖延是因为懒，其实很多时候是因为任务太大，大脑自动开始逃避。当任务看起来很难时，大脑会倾向于选择更轻松的事情，比如刷手机、整理桌面、喝水。解决拖延的关键不是逼自己马上完成全部，而是把任务拆成一个很小的开始。比如先学五分钟，先写一行字，先打开资料。只要开始了，大脑的压力就会下降，继续做下去就容易多了。",
  "ending": "所以，打败拖延最好的办法不是等状态，而是先开始五分钟。"
}
```

## 使用模型

```text
DeepSeek
```

## 设计说明

该智能体决定视频内容质量。
要求输出内容适合配音，句子不能太长，表达要口语化。

---

# 8.3 StoryboardAgent：分镜设计智能体

## 功能定位

StoryboardAgent 负责把完整脚本拆分为多个镜头，并为每个镜头生成时间、口播、字幕和画面描述。

## 输入

```json
{
  "script": "你是不是每次准备学习，都会先刷五分钟手机……",
  "duration": 60,
  "style": "轻松幽默",
  "ratio": "9:16"
}
```

## 处理内容

StoryboardAgent 需要完成：

```text
将脚本拆成 6~12 个镜头
确定每个镜头时长
生成每个镜头的口播内容
生成每个镜头的字幕
生成每个镜头的画面描述
判断哪些镜头适合图生视频
```

## 输出

```json
[
  {
    "scene_id": "scene_001",
    "scene_index": 1,
    "duration": 5,
    "voiceover": "你是不是每次准备学习，都会先刷五分钟手机？",
    "subtitle": "准备学习，却先刷手机？",
    "visual_description": "一名大学生坐在书桌前准备学习，旁边手机不断弹出消息通知",
    "camera_motion": "slow_zoom_in",
    "need_dynamic_video": true
  },
  {
    "scene_id": "scene_002",
    "scene_index": 2,
    "duration": 6,
    "voiceover": "你以为拖延是因为懒，其实很多时候是因为任务太大。",
    "subtitle": "拖延不一定是懒",
    "visual_description": "学生看着满屏任务清单，表情崩溃，画面轻松幽默",
    "camera_motion": "pan_left",
    "need_dynamic_video": false
  }
]
```

## 使用模型

```text
DeepSeek
```

## 设计说明

该智能体是连接文本和视频的关键。
后续的图片生成、图生视频、字幕、剪辑都依赖分镜数据。

---

# 8.4 PromptAgent：视觉提示词智能体

## 功能定位

PromptAgent 负责将分镜中的中文画面描述转化为适合图片生成和视频生成模型使用的提示词。

## 输入

```json
{

# 8.7 VoiceAgent：配音智能体

## 功能定位

VoiceAgent 负责将每个分镜的配音文本转化为独立的配音音频文件。

## 输入

```json
{
  "scenes": [
    {
      "scene_id": "scene_001",
      "voiceover": "你是不是每次准备学习，都会先刷五分钟手机？",
      "duration": 5
    }
  ]
}
```

## 处理内容

VoiceAgent 需要完成：

```text
清洗每个分镜的配音文本（去说话人标签）
为每个分镜单独调用 TTS 生成 voice_{scene_id}.mp3
保存每个分镜配音到 assets 集合（带 scene_id 标记）
更新 scenes 集合中每个分镜的 audio_path 和 audio_duration
同时生成完整的 voice.mp3（用于前端预览）
不再覆盖分镜时长，视频总时长以分镜规划的原始时长为准
```

## 输出

```json
{
  "voice_path": "/assets/task_001/audio/voice.mp3",
  "per_scene": [
    {"scene_id": "scene_001", "path": "/assets/task_001/audio/voice_scene_001.mp3"},
    {"scene_id": "scene_002", "path": "/assets/task_001/audio/voice_scene_002.mp3"}
  ],
  "duration": 58.6,
  "status": "success"
}
```

## 默认方案

```text
edge-tts
```

## 可选升级方案

```text
CosyVoice
```

## 设计说明

第一版建议使用 edge-tts，因为它不花钱，工程实现简单。
VoiceAgent 不再根据配音时长覆盖分镜时长，保证视频总时长以分镜规划为准。
每个分镜生成独立配音文件，供 EditorAgent 按分镜蒙太奇合成。
如果后续对配音效果要求更高，可以切换为阿里 CosyVoice。

# 8.10 EditorAgent：自动剪辑智能体

## 功能定位

EditorAgent 负责将图片、视频片段、配音、字幕和背景音乐合成为最终短视频。

## 输入

```json
{
  "scenes": [
    {
      "scene_id": "scene_001",
      "asset_path": "/assets/task_001/videos/scene_001.mp4",
      "duration": 5,
      "subtitle": "准备学习，却先刷手机？",
      "audio_path": "/assets/task_001/audio/voice_scene_001.mp3",  // 分镜独立配音
      "camera_motion": "slow_zoom_in"
    }
  ],
  "subtitle_path": "/assets/task_001/subtitles/subtitle.srt",
  "music_path": "/assets/music/light.mp3",
  "ratio": "9:16"
}
```

## 处理内容

EditorAgent 需要完成：

```text
统一所有素材尺寸
处理竖屏比例 9:16
将图片转为视频片段
对静态图片添加镜头运动效果
遍历分镜，为每个分镜加载独立配音（scene_audio_map）
将分镜配音挂载到对应 clip 上
拼接所有镜头
添加转场
烧录字幕或外挂字幕
导出 final.mp4
```

## 输出

```json
{
  "video_path": "/outputs/task_001/final.mp4",
  "duration": 59.2,
  "status": "success"
}
```

## 使用技术

```text
MoviePy
FFmpeg
```

## 设计说明

EditorAgent 是系统工程实现的核心。
它不需要调用付费 API，全部使用本地库完成。
EditorAgent 不再使用单一的 voice.mp3 作为整段视频配乐，
而是从 assets 集合中读取每个分镜的独立配音（scene_id 标记），
把配音挂载到对应分镜的 clip 上，避免因配音长度裁剪视频总时长。

## 9. Agent 协作流程

### 9.1 标准生成流程

```text
1. 用户提交一句话需求
2. TaskPlannerAgent 解析任务
3. ScriptAgent 生成脚本
4. StoryboardAgent 生成分镜
5. DialoguePolishAgent 精修对白
6. PromptAgent 生成图片和视频提示词
7. ImageAgent 生成每个镜头图片
8. VideoAgent 对动态镜头生成视频（失败自动降级为静态图）
9. VoiceAgent 生成每个分镜的独立配音（不覆盖分镜时长）
10. SubtitleAgent 生成字幕
11. EditorAgent 合成视频（每个分镜挂载独立配音）
12. QualityAgent 检查视频结果
13. 如果质检失败，返回对应 Agent 重试
14. ExportAgent 导出最终结果
```

---

### 9.2 失败重试流程

```text
图片生成失败
→ ImageAgent 重新生成
→ 若再次失败，使用默认占位图

图生视频失败
→ VideoAgent 重试
→ 若失败，降级为静态图片镜头

配音失败
→ VoiceAgent 重试
→ 若失败，提示用户重新生成

视频合成失败
→ EditorAgent 检查素材
→ 修复素材路径或格式后重新合成

质检失败
→ QualityAgent 给出问题
→ 返回对应 Agent 重新执行
```

# 23.12 前端交互流程

用户完整操作流程如下：

```text
1. 用户进入创建视频页面
2. 输入一句视频需求
3. 选择视频时长、比例、风格和生成模式
4. 点击“开始生成视频”
5. 页面跳转到任务进度页
6. 系统展示各 Agent 执行状态
7. 脚本和分镜生成完成后，用户可进入分镜编辑页查看
8. 素材生成完成后，用户可预览图片和视频片段
9. 视频合成完成后，用户进入视频预览页
10. QualityAgent 展示质检结果
11. 质检通过后，用户下载最终视频
12. 任务保存到历史任务页面
```

# 23.15 前端页面总结

前端页面需要体现系统的完整工作流，而不是只做一个简单输入框。

本系统前端应重点展示：

```text
输入层：用户一句话创建视频任务
过程层：多个 Agent 协作执行
内容层：脚本、分镜、素材、字幕、配音

管理层：历史任务、Agent 日志、系统设置

```text

```

因此，前端不仅是操作界面，也是展示本项目“多智能体协作生成视频”核心价值的重要部分。

# 24.19 错误提示设计

当接口调用失败时，前端需要显示明确错误。

示例：

```text
图片生成失败，请检查通义 API Key 或稍后重试。
```

```text
视频合成失败，可能是素材缺失，请前往质检页面查看详情。
```

错误提示不要只显示：

```text
Error
```

而要显示用户能理解的原因和下一步操作。

# 24.20 前端 UI 总结

本系统前端 UI 设计的核心不是复杂动画，而是清晰展示：

```text
用户输入了什么
系统生成到了哪一步
每个 Agent 正在做什么
每一步生成了什么结果
最终视频是否合格
用户可以如何修改和重新生成
```

因此，前端 UI 应围绕以下五个重点设计：

```text
1. 一句话创建任务
2. 多 Agent 流程可视化
3. 分镜卡片化编辑
4. 素材和视频预览
5. 质检结果和日志可追踪
```

这样可以让用户直观看到：

```text
一句话如何一步一步变成完整短视频。
```
